# Audit 模块规则

## 职责

本模块提供追加写业务审计和安全审计能力，记录可追溯的状态变化，不承担业务决策。

## 不变量

- 审计只追加，不提供普通更新和删除业务接口。
- 审计与对应业务状态变更必须在同一事务边界提交。
- 失败写审计时，要求审计的业务变更必须回滚。
- before/after 快照只保存必要字段，不保存明文凭据、完整请求 Header、Prompt 原文或无关健康数据。
- 审计 action、entityType 和结果语义必须稳定，不以自由文本代替错误码。

## 边界

- 审计模块不查询或推断业务状态，不修复失败业务。
- Agent 运行日志、用户可见解释和领域审计是三种不同用途，不得混成一张无边界日志。
- 调试日志不能替代审计记录。

## 验证

覆盖追加写、JSONB、事务回滚、敏感信息脱敏和重复幂等请求不重复审计。
