# Device 模块规则

## 职责

本模块负责私有单用户的设备身份、首台初始化、后续设备配对、设备凭据、主设备转移、设备撤销和请求认证上下文。

## 不变量

- 不扩展为注册、用户名密码、角色权限、多租户或完整 IAM。
- 首台初始化码只能由服务端 CLI 生成，短时有效、一次性使用，服务端只保存摘要。
- 配对码和二维码内容不得保存到数据库、普通日志或审计快照。
- 每台设备使用独立凭据，服务端只保存摘要。
- 签发凭据的 POST 必须支持安全幂等重放。
- 同一用户任意时刻最多一个 `TRUSTED_PRIMARY`。
- 不得撤销最后一台活跃可信设备。
- 主设备撤销前必须显式转移，不自动选择继任设备。
- `/api/v1/**` 默认要求设备认证，公开白名单必须显式且最小。

## 边界

- 本模块产生 `DevicePrincipal`，业务模块只消费该上下文。
- 不处理 Goal、Plan、健康约束或 Agent Prompt。
- Flutter 负责客户端安全存储；后端不在日志中输出凭据内容。
- 认证错误必须脱敏，不暴露内部堆栈。

## 验证

必须覆盖并发消费、错误次数、过期恢复、幂等重放、凭据轮换、主设备转移、最后设备保护、审计脱敏和认证白名单。
