# Goal 模块规则

## 职责

本模块维护用户目标的唯一事实来源，并为计划关联、进度解释和 Agent 上下文提供稳定身份。

## 当前阶段

- M2.5-A 不改变 Goal 业务语义。
- M2.5-B 可以演进为开放表达，但必须继续使用现有 Goal 作为唯一事实来源。

## 不变量

- 禁止新增并行 `OpenGoal` 或第二套目标事实表。
- 用户原始表达和期望结果是开放目标的核心；固定 GoalType 只能作为可选归一化标签。
- 结构化指标、单位、基线和日期可以为空，不得限制未知目标表达。
- 旧数据不得通过迁移伪造 `originalText` 或 `desiredOutcome`。
- AI 只能创建候选或补全建议，用户确认后才修改目标事实。
- 已完成、已取消和已归档目标保持终态语义。

## 边界

- Goal 不是计划任务，不直接包含每日动作。
- 不在本模块生成 Program、Phase、WeeklyPlan 或 DailyAction。
- PlanVersion 继续关联同一个 Goal，不建立同步镜像。
- 主客户端不要求用户选择内部 GoalType 或技术单位枚举。

## 验证

覆盖状态机、归档、结构化指标兼容、旧数据迁移、单一事实来源以及计划关联稳定性。
