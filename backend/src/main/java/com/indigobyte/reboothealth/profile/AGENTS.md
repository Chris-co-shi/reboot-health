# Profile 模块规则

## 职责

本模块维护用户基础档案和已确认健康约束，作为计划、安全规则和 Agent 上下文的事实来源。

## 当前阶段

- M2.5-A 不改变 UserProfile 或 HealthConstraint 的业务语义。
- 健康约束开放表达和 AI 候选层只允许在明确的 M2.5-B 任务中实施。

## 不变量

- 档案基线体重不是当前体重；当前体重以后从 Observation 或指标记录查询。
- 医疗信息只表达用户陈述、专业建议来源、限制和提醒，不生成诊断。
- AI 不得直接创建、停用、删除或弱化已确认健康约束。
- 重要健康约束必须由用户确认后成为事实。
- 归档不物理删除；归档后禁止普通编辑和状态变更。
- 用户原始陈述不得被归一化标签覆盖或丢失。

## 边界

- 枚举只能作为内部标签，不得成为用户自然表达的边界。
- 不在本模块生成训练计划、每日行动或 Agent Prompt。
- 不把模型置信度等候选信息直接写入已确认事实，除非领域模型明确支持并经过用户确认。

## 验证

覆盖唯一档案、日期和时区、状态转换、归档、审计、健康约束快照以及 AI 无权直接修改已确认约束。
