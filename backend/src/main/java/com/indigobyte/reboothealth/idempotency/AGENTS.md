# Idempotency 模块规则

## 职责

本模块保证关键写请求在网络重试、超时和客户端重复提交时只产生一个业务结果。

## 不变量

- 幂等键必须与 operation 和规范化 request hash 共同校验。
- 同 key、同 operation、同请求返回第一次业务结果。
- 同 key 但请求不同必须明确冲突。
- 重放不得重复状态转换、远程调用或审计。
- 普通幂等记录不得保存明文设备凭据或敏感健康原文。
- 凭据类成功响应只能通过专用受认证加密信封短期重放。
- 幂等记录与业务结果的事务边界必须明确，不能产生已完成记录但无业务结果。

## 边界

- 幂等模块不决定业务是否允许，只保证同一请求不重复执行。
- 不把幂等当作并发控制或 revision 校验的替代品。
- 不无限期保存响应信封；过期和清理策略必须配置化。

## 验证

覆盖首次执行、成功重放、处理中冲突、请求 hash 冲突、失败重试、并发提交、审计次数和敏感响应加密。
