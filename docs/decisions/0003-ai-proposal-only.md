# 0003 AI 候选、确认与低风险自动调整

## 状态

已确认；AI-first 交互和分级确认边界由 ADR 0009 补充。

## 决策

AI 生成理解、目标、健康约束、长期方向、阶段和周计划候选，不能绕过 Java 领域服务直接修改业务事实或发布生效计划。

以下重要变化必须用户确认：

- 长期方向和当前阶段。
- 新的一周计划。
- 增加训练量、强度或复杂度。
- 重要目标变化。
- 新增、弱化或停用重要健康约束。

允许系统自动执行可撤销、可解释、可审计的低风险调整，但只能降低风险或复杂度，例如减少当天训练量、调整顺序或替换为恢复活动。

## 原因

项目涉及健康限制、身体不适和训练安全，模型不能成为最终业务权威。同时，所有微小调整都要求手工确认会使日常体验失去价值，因此采用分级权限。

## 影响

- AI 输出必须经过结构校验。
- 确定性安全规则优先于建议应用。
- 用户确认后的计划由 Java 计划领域服务创建或发布新 PlanVersion。
- 自动调整必须记录原因和前后差异，并允许撤销。
- AI 不得自动提高训练风险或弱化已确认健康约束。

## OPEN

- `OPEN`：真实模型供应商和模型。
- `OPEN`：自动调整的产品展示和撤销时限。
