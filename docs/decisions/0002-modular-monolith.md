# 0002 模块化单体架构

## 状态

已确认

## 背景

项目是单人使用的健康管理和训练辅助应用，当前目标是尽快打通计划、执行、分析、AI 建议、用户确认和计划版本生成的闭环。部署目标是私有 PC、Docker 和宿主机 PostgreSQL。

## 决策

MVP 采用模块化单体，不拆微服务。

后端模块包括：

- `profile`
- `goal`
- `plan`
- `execution`
- `metrics`
- `analysis`
- `rules`
- `ai`
- `adjustment`
- `audit`

## 备选方案

- 微服务：边界更强，但对个人项目过度复杂，会增加网络、部署、观测和一致性成本。
- 单包 CRUD：初期文件少，但计划版本、规则和 AI 调整边界容易混杂。
- 模块化单体：保持部署简单，同时能表达领域边界。

## 影响

- 一个后端应用。
- 一个数据库。
- 领域聚合负责关键状态转换。
- 模块依赖方向需要持续维护。
