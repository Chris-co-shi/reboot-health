# 个人健康、减脂与体能重建应用 MVP 规划

## 1. 产品定位

本项目是个人使用的健康管理和训练辅助工具，服务目标是帮助用户在减脂、心肺恢复、力量和肌耐力重建过程中记录数据、执行计划、复盘进度，并在安全规则约束下使用 AI 生成调整建议。

本项目不做医学诊断，不替代医生意见。系统只提供记录、提醒、计划辅助和风险提示。

## 2. 用户情况

- 男，34 岁。
- 身高 175cm。
- 当前体重约 93-95kg，目标约 80kg。
- 腹部脂肪增加。
- 最大力量、肌耐力、心肺体能下降。
- 篮球两个回合后明显喘。
- 游泳换气困难，容易呛水。
- 医生建议进行游泳训练。
- 有高血压，正在按医嘱服用降压药。
- 有多节段颈椎间盘突出及轻度退行性改变。
- 有肩颈、髋、腰、足底和跟腱方面的限制。

## 3. MVP 闭环

1. AI 起草一周计划。
2. 用户查看计划草案、风险提示和可调整项。
3. 用户确认后生成 `PlanVersion v1`。
4. 今日页展示当天任务。
5. 用户记录训练执行、身体指标、症状、睡眠、疲劳和饮食执行。
6. 系统生成周分析。
7. 确定性规则先评估安全边界。
8. AI 生成结构化调整建议。
9. 系统校验 JSON Schema 和安全规则。
10. 用户全部接受、部分接受或拒绝。
11. 接受项生成新的 `PlanVersion`。

## 4. MVP 页面

1. 今日：展示当天计划、完成状态、关键提醒。
2. 当前计划：展示当前计划版本、一周安排、健康约束摘要。
3. 数据记录：录入训练、身体指标、睡眠、疲劳、疼痛和饮食执行。
4. 趋势分析：展示周分析、趋势图和规则警告。
5. 调整确认：展示 AI 建议、前后差异、依据、风险和用户确认操作。

## 5. 非目标

- 不做登录、注册、权限、多租户。
- 不做社区、排行榜、教练商城。
- 不做支付。
- 不做食物拍照识别。
- 不做动作视频识别。
- 不做 Apple Health 或手环同步。
- 不做原生 App。
- 不拆微服务。
- 不自动生成全部长期训练内容。
- 不做自动医学诊断。

## 6. 核心领域对象

- `UserProfile`：个人档案。
- `HealthConstraint`：健康约束和运动限制。
- `Goal`：目标体重、训练目标、阶段目标。
- `Plan`：计划主体。
- `PlanVersion`：计划版本。
- `PlanDay`：计划中的某一天。
- `PlanItem`：某天的训练、恢复、饮食或记录任务。
- `DailyLog`：每日执行摘要。
- `TrainingSession`：训练记录。
- `BodyMetricEntry`：身体指标记录。
- `SymptomEntry`：疼痛或不适记录。
- `WeeklyAnalysis`：周期分析快照。
- `AdjustmentProposal`：AI 调整建议。
- `AdjustmentDecision`：用户确认结果。
- `AuditLog`：审计记录。

## 7. 关键不变量

- AI 不能直接发布生效计划。
- AI 不能删除健康约束。
- 生效计划不能原地修改，只能创建新版本。
- 历史执行记录不得随计划版本变化而改变。
- AI 输出必须先通过 JSON Schema 校验。
- 被确定性规则阻断的建议不能应用。
- 疼痛、睡眠不足、游泳呛水、血压异常等安全因素必须由程序规则处理。
- 医疗警告只能表达为提醒，不能生成诊断结论。

## 8. 记录粒度

采用按项目分层：

- 徒手训练：记录动作、组数、次数、RPE、疼痛评分、备注。
- 游泳训练：记录时长、距离、泳姿、呛水次数、换气困难评分、RPE、备注。
- 篮球训练：MVP 先记录时长、强度、喘息评分、备注。
- 健身房训练：记录器械、重量、组数、次数、RPE、疼痛评分。
- 饮食执行：MVP 先记录执行度和备注，不做食物库和热量核算。

## 9. 里程碑

### M1 文档和工程骨架

范围：创建文档、后端空骨架、前端空骨架和部署配置。

验收：

- 文档明确 MVP 边界和安全不变量。
- 后端健康检查可用。
- 前端 5 个页面占位可访问。
- Docker Compose 配置可校验。

### M2 档案、约束、目标和 AI 起草计划

范围：录入个人档案、健康约束、目标，调用 AI 起草一周计划，用户确认后生效。

验收：

- 计划草案不能直接生效。
- 用户确认后生成第一个计划版本。
- 审计记录可追溯。

### M3 今日页和每日记录

范围：展示今日任务，录入完成情况和分层记录。

验收：

- 每日记录关联当前计划版本。
- 可记录训练、睡眠、疲劳、疼痛、血压和饮食执行。

### M4 周分析和规则引擎

范围：生成 7 天分析，执行确定性安全规则。

验收：

- 输出完成率、体重、血压、睡眠、疲劳、疼痛趋势。
- 规则结果包含 `ALLOW`、`WARN`、`BLOCK`。

### M5 AI 调整建议

范围：真实 OpenAI 兼容接口、JSON Schema 校验、建议保存。

验收：

- 无效 JSON 被拒绝。
- 建议项包含依据、风险和前后差异。
- 被规则阻断的建议不可进入可应用状态。

### M6 调整确认和计划版本生成

范围：全部接受、部分接受、拒绝，生成新计划版本。

验收：

- 只有用户确认的建议能应用。
- 旧执行记录不变。
- 新旧计划差异可查看。

### M7 部署、备份和验收

范围：Tailscale 访问说明、本地导出备份、端到端验收。

验收：

- 可在 Windows 10 + Docker 环境运行。
- 可导出健康数据。
- 完整闭环可人工验收。

## 10. 安全默认值

MVP 使用保守默认阈值，后续可按医嘱调整。阈值只用于提醒、降级和阻断训练升级，不用于诊断。

初始规则方向：

- 疼痛评分达到阈值时，禁止增加相关部位负荷。
- 连续睡眠不足时，禁止升级训练量。
- 游泳呛水次数增加时，禁止自动增加距离或强度。
- 训练完成率过低时，优先降低复杂度。
- 血压异常时，提示复测和咨询医生。
