# 部署目录规则

本目录只负责 Docker Compose、容器构建、环境变量示例、端口绑定和健康检查。

- 不在部署任务中顺带修改业务代码。
- 默认绑定本机安全地址，不擅自改为公网或局域网全开放。
- 仓库只保存示例配置，不保存真实数据库地址、密码或加密材料。
- 环境变量名称必须与应用配置一致。
- Agent Runtime、Backend 和调试前端的依赖关系必须明确。
- Vue 是内部调试工具，不作为正式客户端扩展。
- 不自动修改宿主机 PostgreSQL、操作系统防火墙或 Tailscale 配置。
- 增加服务时必须提供健康检查、启动依赖和失败说明。
- M2.5-A 不引入 Redis、消息队列、向量数据库或额外工作流服务。

验证：

```bash
docker compose -f deploy/docker-compose.yml config
```
